# BACKLOG — v1.0 スコープ外項目

このドキュメントは v1.0 (2026-10-28 リリース) では実装しないが、将来再評価する項目を保管する。
v1.0 リリース後に優先順位を見直すこと。

## 認証・認可

### Entra ID (Azure AD) 完全統合
- **却下理由**: v1.0 期間内では SCIM プロビジョニング/SSO 設計に 1 ヶ月以上必要
- **代替**: v1.0 では Phase C-3 で AD 基本連携 (ユーザー同期のみ) を行う
- **再評価**: v1.1 マイルストーン

### 10 ロール権限体系
- **却下理由**: 中小規模 IT 部門の実運用では admin/operator/viewer の 3 段階で過不足なし
- **代替**: v1.0 は 3 ロール体系を維持
- **再評価**: 顧客フィードバックで「より細かい権限分離が必要」となった場合

## 自動化

### AI 自動修復 (自律バグ修正)
- **却下理由**: 監査要件 (J-SOX / ISO 27001) と相反する可能性。承認なし自動変更は監査ログ要件と整合困難
- **代替**: Claude Code との対話的修復にとどめる
- **再評価**: 監査要件を満たす設計が整理できた時点

### キッティング自動化 (Windows 初期構築)
- **却下理由**: スコープが OS デプロイ領域に踏み込む。MDT/Intune が既存ソリューションとして存在
- **代替**: v1.0 では Agent 配布スクリプトのみ提供
- **再評価**: 顧客から強い要望があった場合

## 資産管理

### QR コード資産インベントリ
- **却下理由**: ハードウェア (ラベルプリンタ/スキャナ) を含む業務プロセス変更が必要
- **代替**: v1.0 ではシリアル番号ベース管理
- **再評価**: 物理棚卸自動化の要望が出た場合

### PC ライフサイクル全機能
- **却下理由**: 廃棄ワークフロー・リース管理・減価償却は会計システム連携が必要
- **代替**: v1.0 では「導入日」「更新予定日」のみ管理
- **再評価**: 会計/購買システム連携要件が固まった時点

## 監視・分析

### 機械学習による異常検知
- **却下理由**: 学習データ・モデル運用基盤が未整備
- **代替**: v1.0 では閾値ベースのアラートエンジン (Phase C-1)
- **再評価**: v1.0 でメトリクス蓄積が進んだ時点
